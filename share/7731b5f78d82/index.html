⚠️ რა გავასწორე (გულახდილად)

ჩემი შეცდომა: ძალიან მივაჭირე „დაკეტვას" და დიასპორა მთელ მისიად ვაქციე. შენი ორი-აუდიტორიის ხედვა (საქართველოს ბავშვები — ინგლისი/მათემატიკა/განვითარება) ჩავკუმშე და დავკარგე. შენი brief ჩემს კონტექსტში არ იყო — მხოლოდ შეკუმშული შეჯამება. ახლა docs/OWNER_GRILLDOWN_BRIEF.md-ში ზუსტად ინახება.
Spearhead ≠ მისია. დიასპორა = სად არის პირველი ფასიანი დოლარი ყველაზე ადვილი. ეს არ ნიშნავს რომ საქართველოს ბავშვებს ვტოვებთ.
გასწორებული მისია (ორივე აუდიტორია):
„რომ ყველა ქართველმა ბავშვმა — საქართველოში თუ მის გარეთ — მიიღოს უსაფრთხო, ხარისხიანი სივრცე, სადაც ისწავლის იმას, რაც სჭირდება (ინგლისი/მათემატიკა სახლში · ქართული/იდენტობა საზღვარგარეთ) და განვითარდება."

ეს დოკ. = PLAN. არაფერი არ არის „locked". ყველაფერი შენი რეაქციისთვისაა. წინა „სახელმძღვანელო" (over-locked) ამით იცვლება.

🔥 უხერხული სიმართლე (grilldown)

1. „ორმაგი მისიის" პარადოქსი 🔷
solo-დამფუძნებელი ვერ აჭერს ორ მაღალი დონის კურიკულუმს. რისკი: ორივესთვის „ზედაპირული" → მშობელი 3 დღეში შლის. 🦉 პასუხი: ერთი ძრავა, ორი ლინზა (იხ. ⚙️ ტაბი) — ერთი content-ი ორივეს ემსახურება.
2. „უსაფრთხო გარემო" ≠ USP 🔷
უსაფრთხოება baseline-ია, არა გასაყიდი ღირებულება. მშობელი იხდის გაზომვად პროგრესში: „ჩემმა ბავშვმა ქართული წინადადება წაიკითხა" > „უსაფრთხოა". შენი საყვარელი positioning საჭიროა, მაგრამ ქვეშ პროგრესი უნდა ედოს.
3. AI თეატრი 🔷
ახლანდელი „tutor" = წესების hint-ძრავა, არა AI. მშობელს „AI"-ად მისი მიყიდვა გაგვიბრუნდება. ნამდვილი AI = content-pipeline ან feedback (იხ. 🤖 ტაბი).
4. „ყველას ვემსახურები" ხაფანგი 🦉
ორი აუდიტორია სწორია მისიად — მაგრამ go-to-market-ში თანმიმდევრობა გჭირდება. ერთდროულად ორ ბაზარზე მარკეტინგი solo-სთვის თვითმკვლელობაა. ერთით იწყებ ფასიანს (დიასპორა, მაღალი WTP), მეორეს უფასოდ ზრდი.
5. 6-თვიანი შედეგი = პასუხისმგებლობა 🔷
„Cambridge ქულა 6 თვეში" — თუ ვერ მიაღწიე, liability. გადაი „საფუძველზე": თავდაჯერება · ყოველდღ. რუტინა · სკოლისთვის მზაობა (უხილავი კურიკულუმი).

⚙️ ერთი ძრავა, ორი ლინზა — გასაღები

ნუ ვტოტავთ პროდუქტს. ერთი ძრავა, „სამიზნე ენის" გადამრთველი.
აუდიტორიაUI ენა (იცის)სამიზნე (სწავლობს)
🇬🇪 საქართველოქართულიინგლისური + მათემატიკა
🌍 დიასპორაინგლისური/ადგ.ქართული + იდენტობა

მშობელი setup-ზე ირჩევს სამიზნე ენას. ძრავა ცვლის რომელი მხარეა „კითხვა" და რომელი „პასუხი". ერთი content-pair (KA / EN / emoji / audio) ორივე მიმართულებით მუშაობს. 🔷

content-იერარქია (შენი „უდიდესი უცნობი")

Skill (კითხვის სრბ., ლოგიკა, ლექსიკა) → Module („ტყე", „სკოლაში") → Content-Pair (KA/EN/emoji/audio) → Directional logic (რომელი მხარეა prompt/target).
ეს = content-agnostic engine, რომელზეც ვსაუბრობდით — ოღონ ახლა ორ აუდიტორიას ემსახურება, არა მხოლოდ მომავალ ენებს.

ასაკი vs უნარი (grades 1-3)

verdict: cognitive-capacity. ასაკი = მოტორიკა/ყურადღება; უნარი = სირთულე.
პატარებს „რელსი" (auto-progression): მენიუ კი არა, ერთი „დღევანდელი თავგადასავალი" ღილაკი. 3 სწორი = ზემოთ · 5 შეცდომა = ქვემოთ + მშობელს რეპორტი. მენიუ მშობლისთვისაა, რელსი ბავშვისთვის. 🔷

🤖 AI — ნამდვილი moat vs theater

თეატრი (ყალბი moat): AI ხმოვანი ავატარი, ბავშვს რომ პასუხს აძლევს. ბავშვის მეტყველება ცვალებადია; latency/შეცდომა 6 წლისას გააღიზიანებს.
ნამდვილი moat #1 — AI content-pipeline: LLM-ით ათასობით „Kings-სტილის" ამოცანა და ქართული საკითხავი ტექსტი offline გენერდება და აპში „ჩაცხვება". ეს wგვარს solo-დამფუძნებლის content-მოცულობის პრობლემას.
ნამდვილი moat #2 — „მშობლის თარჯიმანი": AI ნედლ „20/25 სწორი"-ს მშობლისთვის გასაგებ ტექსტად აქცევს: „ნიკომ ცხოველები აითვისა, მაგრამ ინგლისურ 'th' ბგერაზე უჭირს. დღეს [ფრაზა] ერთად წაიკითხეთ." ეს არის „მშობლის სიმშვიდის" პროდუქტი. 🔷

დასკვნა: AI-ს არ ვყიდით როგორც feature-ს. AI = backend-ფაბრიკა (content) + insight-ძრავა (მშობელი). ეს უხილავია და სწორედ ამიტომ moat.

🅰️ Version A — შენი ხედვის გახვეწილი

„ორმაგ-მისიის" ბილინგვური პლატფორმა

არსი: ერთი პროდუქტი, ერთი ძრავა, ორი ლინზა. ორივე აუდიტორია შიგნით.

მოდელი: freemium. ბირთვი უფასო; „Kings-მზადება" + „გაღრმავ. ქართული" + მშობლის insight = subscription. დიასპორა = პირველი ფასიანი wedge.

პოზიცია: „უსაფრთხო, ხარისხიანი სივრცე, სადაც ქართველი ბავშვი სწავლობს იმას, რაც სჭირდება — სახლში თუ საზღვარგარეთ."

დიფერენციაცია: ერთადერთი აპი, რომელიც ბილინგვურ ქართულ ოჯახს ორივე მიმართულებით ერგება + ნამდვილი ხმა + კურირებული უსაფრთხოება + ნიკო ბუ.

რისკი: content-დატვირთვა ორ მიმართულებაზე; „best-in-class" ვერც ერთში; საქართველოში ინგლისზე უფასო გიგანტებთან შეჯახება.

retention: ყოველდღ. რელსი + მშობლის dashboard + streak.

მონეტიზაცია: subscription (დიასპ.) + freemium-upsell (GE).

scale: content-agnostic engine → მეტი მოდული, მერე სხვა heritage ენები.

🅱️ Version B — დამოუკიდებელი, უფრო ძლიერი

„უნარ-ძრავა ჯერ" (skill-engine first)

არსი: ბირთვი = skill-based ადაპტური ძრავა (რელსი + მშობლის თარჯიმანი + AI content-pipeline). საგნები (ინგლ./ქართ./მათ.) = content-მოდულები ამ ძრავაზე.

მოდელი: subscription insight/advanced ფენაზე. (B2B2C = არჩევითი დამაჩქარებელი.)

პოზიცია: „იცოდე ზუსტად რა ისწავლა შენმა ბავშვმა და რა ქნა შემდეგ" — მშობლის სიმშვიდე + გაზომვადი პროგრესი, ნებისმიერ ენაზე.

დიფერენციაცია: ძრავა + „მშობლის თარჯიმანი" = ნამდვილი moat, ყველაზე ძნელად საკოპირებელი, ყველაზე დაბალი content-treadmill solo-სთვის.

რისკი: „უნარი/cognitive" უფრო ძნელი გასაყიდია ვიდრე „ინგლისი"; ბრწყინავს მხოლოდ ძრავის აშენების შემდეგ.

retention: ყველაზე მაღალი (ადაპტური + მშობლის insight-მარყუჟი).

მონეტიზაცია: subscription insight-ფენაზე.

scale: საუკეთესო — ძრავა + pipeline იაფად გადადის ნებისმიერ საგანზე/ენაზე.

⚖️ Tradeoff + ჩემი რეკომენდაცია (შენი call, არა locked)

A დღეს უფრო ადვილი გასაყიდია (მშობელს ესმის „ინგლისი", „მათემატიკა"). B უფრო ძლიერი long-term moat solo-სთვის (სისტემა > content-treadmill).

🦉+🔷 სინთეზი: აიღე Version A-ის ლეიბლები (ინგლისი/ქართული/მათემატიკა — რასაც მშობელი ხვდება და იხდის), მაგრამ ააშენე Version B-ის ძრავა (skill-based, ადაპტური, მშობლის თარჯიმანი, content-pipeline). GTM-ში დაიწყე დიასპორით (პირველი ფასიანი), საქართველო პირველივე დღიდან შიგნით იმავე ძრავაზე. ორივე აუდიტორია რჩება.

❓ ღია — შენი რეაქცია

აღარაფერია „locked". ეს არის ის, რაზეც შენი აზრი მჭირდება.
