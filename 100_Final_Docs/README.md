- **CONDITIONAL-GO როგორც ვალიდაციის სისტემა** (E1 სერვის-ფორმატში, წინასწარი kill-რიცხვებით): **~80%** [RECOMMENDATION]. იცვლება არა „ვაკეთებთ თუ არა", არამედ „რას ვამოწმებთ ჯერ": გადახდას და სწავლის ეფექტს.
- **E1 დღეს გაუზომვადია — ტექნიკური ფაქტი** (B აუდიტი): ინგლისურ სიტყვებზე pre/post დღესვე შესაძლებელია, ქართული ასოების per-item ისტორია კი არ არსებობს. განბლოკვა = ასო-ჩანაწერები + 5-წუთიანი pre/post ზონდი.
- **UI/UX აუდიტი (A):** 4 დადასტურებული დეფექტი = NB-72..75 (ლოკალიზაციის შერევა, მოძველებული owl-overlay, „71 ბარათი" vs „1/54", „სიზუსტე 0%" პირველ დღეს). ფიქს-ტალღა შენი preview-ვერდიქტის შემდეგ.

## ხარისხის ფილტრის შედეგი (research-quality სქილი, 07-17)
Brief v2 გატარებულია 8-კარიბჭიან ფილტრში + red-team. ყველა დამტვირთავი მტკიცება წყაროებში ზუსტად დგას (Q2=9). ორი კარიბჭე პირველ გავლაზე ჩავარდა და გასწორდა:
- **Red-team-ის მთავარი დარტყმა:** „ორი დამოუკიდებელი ოპონენტი" ორივე AI-მოდელია. კონვერგენცია შეიძლება საერთო prior-ის ანარეკლი იყოს („ჯერ სერვისით დაავალიდირე" გავრცელებული სტარტაპ-ორთოდოქსიაა) და არა ორი დამოუკიდებელი დასტური.
- **კალიბრაცია:** მთელ კორპუსში ჯერ არ არის არცერთი რეალური მომხმარებლის მონაცემი: არც მშობლის ინტერვიუ, არც გადახდის ტესტი. ყველა % მოდელის შეფასებაა; პირველივე რეალური მონაცემი მათზე მეტს იწონის.
- **რატომ აძლიერებს ეს დასკვნას:** თუ ოპონენტების რჩევა prior-ია, „ჯერ რეალურ ადამიანებზე ვალიდაცია, E1 hold" კიდევ უფრო სწორია. რეკომენდაცია უცვლელია: სერვის-ფრეიმი, default = hold.
